🖥️ 命令类报错

原因：系统 PATH 环境变量中未包含该软件的执行路径。

解决方案：

  • 确认软件已正确安装
  • 将软件安装目录添加到系统 PATH 环境变量
  • 添加后重启终端窗口再试

详细步骤请查看 环境搭建 → Node.js 环境变量配置

原因：命令未安装，或未添加到 shell 的 PATH 中。

解决方案：

  • 检查是否已通过 npm / brew / apt 正确安装
  • 检查 ~/.bashrc 或 ~/.zshrc 中的 PATH 配置
  • 执行 echo $PATH 查看当前路径

📦 依赖类报错

原因：默认 npm 源在国外，国内网络访问不稳定。

解决方案：配置国内镜像源加速：

npm config set registry https://registry.npmmirror.com

或使用 cnpm：

npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install 包名

原因：PyPI 官方源在国内访问速度较慢。

解决方案：使用国内镜像源：

# 临时使用清华镜像
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名

# 永久配置
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

原因：npm 依赖版本冲突。

解决方案：

  • 使用 --legacy-peer-deps 参数：
    npm install --legacy-peer-deps
  • 或清理缓存后重试：
    npm cache clean --force
rm -rf node_modules package-lock.json
npm install

原因：未安装所需依赖包，或在错误的 Python 环境中运行。

解决方案：

  • 安装缺失包：
    pip install 包名
  • 确认是否在正确的虚拟环境中运行：
    # Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate

🔑 鉴权类报错

原因：API Key 错误、过期、或格式不正确。

解决方案：

  • 检查 API Key 是否复制完整（注意不要有多余空格或换行）
  • 登录官方控制台确认 Key 状态是否有效
  • 确认未超出 API 调用额度
  • 如有疑问，请自行联系官方客服

本站不提供 API Key 代申请、代充值服务。

原因：账号权限不足或 API Key 未被授权访问该模型/资源。

解决方案：

  • 确认账号是否有权访问所请求的模型（如 GPT-4 需要特定订阅）
  • 检查 API Key 的权限范围设置
  • 联系官方平台确认账号状态

🌐 连接类报错

原因：网络环境无法正常访问 API 服务器。

解决方案（自查网络）：

  • 检查本地网络是否正常（能否打开其他网站）
  • 如使用公司/学校网络，检查是否有防火墙限制
  • 如需要特殊网络环境，请用户自行依法办理

本站不提供网络代理、VPN 相关服务或指导。

原因：DNS 无法解析 API 域名。

解决方案：

  • 尝试切换 DNS（如使用 114.114.114.114 或 8.8.8.8）
  • 刷新 DNS 缓存：
    # Windows
ipconfig /flushdns

# Mac
sudo dscacheutil -flushcache

# Linux
sudo systemd-resolve --flush-caches

🔒 系统权限报错

原因：当前用户没有足够的文件系统权限。

解决方案：

  • Windows：以管理员身份运行 PowerShell 或 CMD
  • Mac/Linux：使用 sudo 执行命令，或配置 npm 全局路径避免使用 sudo：
    mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
# 然后将 ~/.npm-global/bin 添加到 PATH

原因：杀毒软件误判 npm/node 为可疑程序。

解决方案：

  • 暂时关闭实时保护（安装完成后重新开启）
  • 将 Node.js 安装目录添加到杀毒软件白名单
  • 从官方网站下载，确保文件来源安全

🔄 版本兼容报错

原因：Claude Code / Codex 等工具要求 Windows 10 以上系统。

解决方案：

  • 升级到 Windows 10（至少 1903 版本）或 Windows 11
  • 如果无法升级，可尝试较旧版本的 Node.js（不保证全部功能可用）

原因：当前 Node.js 版本低于工具要求的最低版本。

解决方案：升级 Node.js 到 LTS 版本：

# 查看当前版本
node --version

# 安装 nvm（Node 版本管理器）来切换版本
# Windows: 下载 nvm-windows
# Mac/Linux: curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装并使用 LTS 版本
nvm install 20
nvm use 20

原因：系统 Python 版本过低（如 Python 2.7）或未安装。

解决方案：

  • 安装 Python 3.10+（推荐）
  • 使用 python3 而不是 python（Linux/Mac）
  • 使用虚拟环境隔离不同项目依赖
